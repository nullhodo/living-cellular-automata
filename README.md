# Living Cellular Automata

Living Cellular Automata(リビング・セル・オートマタ)は、p5.jsを用いて構築された「Cyclic Cellular Automata(巡回型セル・オートマトン)」のウェブブラウザ向けシミュレーションです。カスタマイズ可能なグリッド上で、隣接するセルの状態に応じて各セルが状態を変化させ、まるで生き物のように動的で美しいパターンを描き出します。

## 主な機能

- **リアルタイムシミュレーション**: なめらかでインタラクティブなセル・オートマトンのシミュレーションを提供します。
- **パラメータのカスタマイズ**: セルのサイズ、変化に必要な隣接セルの数(しきい値)、近傍の範囲(半径)、状態の数(色数)、突然変異の確率(ノイズ)、そしてシミュレーションの更新速度を細かく調整できます。また、各パラメータごとに個別のランダム化も可能です。
- **近傍判定の形状選択**: 近傍判定のアルゴリズムを「Square(四角形)」と「Circle(円形)」から選択可能です。円形モードではRangeを0.1刻みで細かく調整できます。
- **カラーパレット & グラデーションモード**: 厳選された美しいプリセットパレットから選択できるほか、「Gradation Mode」を有効にすることで始点・終点の色と分割数(2-7)を指定してグラデーションパレットを動的に生成できます。
- **高画質エクスポート**: 現在のシミュレーション状態を、レイアウトを崩さずに高解像度の画像として書き出せます。
- **動画録画機能**: ブラウザ上から直接、シミュレーションの様子をWebM形式の動画として録画・保存できます。
- **状態管理**: Undo/Redo(元に戻す/やり直し)機能や、JSONCファイルを用いた設定の保存と読み込み(ドラッグ&ドロップ)に対応しています。

## 使い方

1. ウェブブラウザで `index.html` を開きます。
2. 画面右側のツールウィンドウを使って、各種パラメータを調整します。
   - **Gradation Mode(グラデーションモード)**: 有効にすると、プリセットパレット選択が非表示になり、グラデーションの開始色・終了色を選択する2つのカラーピッカーと、色数を指定する「Gradient Colors」スライダーが表示されます。
   - **Neighborhood Type(近傍判定の形状)**: 近傍の判定アルゴリズムを選択します。Square(四角形)はクラシックな挙動となり、Circle(円形)は小数点単位での滑らかな範囲調整が可能です。
   - **Cell Size / Threshold / Range / States**: 巡回型セル・オートマトンの解像度や状態遷移ルールを調整します。
   - **Noise**: グリッドにランダムな突然変異を追加し、パターンの完全な停止を防ぎます。
   - **Speed**: 更新速度を調整します(0で一時停止)。1目盛りあたりの実速度が以前より4倍高速化されています。
   - 各パラメータは個別の「ランダム化」ボタンでランダムな値に変更できます。
3. **高解像度画像書き出し** ボタンで画像をエクスポートするか、**録画開始** ボタンで動画をキャプチャします。
4. 保存された `.jsonc` ファイルをウィンドウ上にドラッグ&ドロップすることで、過去の設定(近傍の判定形状やグラデーション設定を含む)を自動で判別して復元できます。

## プロジェクト構成

- `index.html`
- `style.css`
- `sketch.js`: p5.jsのメインループ処理。
- `js/`: 各機能ごとに分割されたJavaScriptファイルを格納。
  - `config.js`: 初期設定とカラーパレットの定義。
  - `ca.js`: セル・オートマトンのシミュレーションロジック。
  - `ui.js`: ツールチップやカスタムドロップダウンを含むUI制御。
  - `export.js`: 画像および動画の書き出し処理。
  - `history.js`: Undo/Redoによる履歴管理機能。

## 使用技術

- HTML5, CSS3, JavaScript
- [p5.js](https://p5js.org/): キャンバス描画とシミュレーションロジックに使用。

## アルゴリズム

現在実装されているセルオートマトンは、「循環型セルオートマトン(Cyclic Cellular Automaton)」 と呼ばれるアルゴリズムをベースに動いています。

具体的には、以下のルールで各セルが次の状態(色)を決定しています。

1. 状態のサイクル(States)
   各セルは 0 から activeStates - 1 (設定した状態数とパレットの色数の最小値) までの数値(状態)を持っており、それぞれがパレットの色に対応しています。状態は 0 → 1 → 2 → …… → (activeStates - 1) → 0 というように、「自分より1つ上の状態」にのみ進化(変化)しようとします。

2. 観測範囲(Range)と近傍判定
   各セルは、自分の周囲 Range(半径)の距離にあるセルを確認します。判定方法は「Neighborhood Type」の設定によります。また、画面の端と端は繋がっています(トーラス構造)。
   - **Square(四角形)**: クラシックな判定方法。 `max(|dx|, |dy|) <= Range` の正方形範囲のセルを近傍とみなします。
   - **Circle(円形)**: ユークリッド距離に基づく判定方法。 `dx^2 + dy^2 <= Range^2` の円形範囲のセルを近傍とみなします。これによりRangeを小数点単位で微調整できます。

3. 変化の条件(Threshold)
   現在の状態が N であるセルは、周囲の近傍セルのうちに状態 N + 1 のセルが Threshold個以上存在するかを数えます。
   - Threshold 個以上ある場合: 自分の状態を N + 1 に進めます。
   - Threshold 個未満の場合: 変化できず、現在の状態 N を維持します。
